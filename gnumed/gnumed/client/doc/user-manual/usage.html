Chapter 4. How to use GnuMed to Make a Cup of Tea & Other Things (aka Newbie/Etiquette Guide)

We actually haven't quite mastered the tea making. We can however give you a few tips on process. :-) If you keep the following things in mind you'll make life easier for us all:

  • Tell us about all the glorious code you've written. We really do want to hear about it. The place to do this is the mailing-list. If your code is really glorious & important write about it in the documentation (so others can understand the pertinent details quickly).
  • Ask questions. Can it be done better? Can we avoid a future problem? The only caveats are (1) stupid questions get stupid answers and (2) sometimes by trying to figure-out something on your own you learn other (not directly related but useful) things. The best place to ask questions is of course the mailing-list.
  • Keep the code on CVS working. Nobody likes it when they update their source tree, find-out GnuMed doesn't work and then realize GnuMed suddenly has almost as many bugs as a Microsoft product.
  • Many of us have ICQ. My ID is 97418709. Introduce yourself. A two minute conversation on ICQ can sometimes save you a lot of time.
  • Write lots of code and documentation... have fun!

If you're really stuck let us know with a post to the mailing-list that has a copy of your log file (gnumed.log) attached. If you're having a problem chances are someone else is too.

Generally, it is a good idea to run gnumed the following way: 

python gnumed.py --debug
This makes GnuMed generate lots of messages and makes debugging easier. ;-)
